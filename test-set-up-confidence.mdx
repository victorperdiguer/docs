title: "Test Set-up Confidence"
---

The **Test Details** tab is the record of how the test was set up — starting with a read on how sound the configuration was, then the mission, the audience, and the AI participants behind the run.

## Set-up confidence

At the top, Uxia shows confidence indicators for four parts of the setup, so you can gauge how much to trust the results before you act on them:

| Indicator | What it reflects |
| --- | --- |
| **Test Setup** | How well-formed the overall test configuration is |
| **Instructions** | How clear and actionable the mission and scenario are |
| **Designs** | How suitable the uploaded designs or source are for testing |
| **Audience** | How well the audience matches what you set out to test |

When any of these is low, tighten it before reading too much into the findings — a weak mission or an ill-matched audience quietly weakens everything downstream.

## Mission & scenario

The tab shows the original mission and scenario used for the test. For example: _buy a 1-hour standard travel ticket in the GVB app using Mastercard and request a receipt._

## Audience

It also summarizes who the test ran with:

- Audience name
- Tester count
- Age range
- Gender mix
- Filter summary

## AI participants

Finally, the tab lists the **AI Participants**, each with a richer profile so you can see who the synthetic testers were meant to represent:
