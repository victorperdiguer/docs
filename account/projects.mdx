In Uxia, a Project is a way to organize your research work around a specific initiative, client, product area, campaign, or user journey.

As teams run more tests, it becomes important to keep research organized and easy to navigate. A Project acts as a folder inside your workspace where you can group related tests together. This helps teams avoid having all tests mixed in one place and makes it easier to find, compare, and revisit research over time.

## Projects help you organize tests around real work

A Project can represent anything your team is working on. For example, you might create a Project for:

- A new feature or flow
- A client project
- A sprint
- A specific market, audience, or business unit

Inside each Project, you can add the tests that belong to that initiative. This gives your team a cleaner structure and makes it easier to understand which tests are connected to the same research goal.

## Example

Imagine your team is redesigning the onboarding experience for a banking app. You could create a Project called **New Onboarding Flow**.

Inside that Project, you might run:

- An AI User Test on the first prototype
- A second test after design changes
- An Accessibility Test to check usability barriers
- An AI User Research test focused on user expectations or comprehension

All of these tests belong to the same initiative, so grouping them inside one Project makes the research easier to manage and review.
